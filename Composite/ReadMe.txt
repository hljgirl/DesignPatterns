组合

将对象组合成树形结构以表示“部分-整体”的层次结构.Composite使得用户对单个对象和组合对象的使用具有一致性.




结构
1. Component(Employee):组合对象的抽象.
2. Leaf(ShopSaler):组合对象节点.
3. Composite(ShopKeeper):含有组合对象的节点对象.

优点
1. 定义了包含基本对象和组合对象的类层次结构.
2. 简化客户代码.
3. 使得更容易增加新类型的组件.
4. 使你的设计变得更加一般化.

缺点
1. 使设计变得更加抽象，对象的业务规则如果很复杂，则实现组合模式具有很大挑战性，而且不是所有的方法都与叶子对象子类都有关联.
2. 需要考虑”透明方式”和”安全方式”.
